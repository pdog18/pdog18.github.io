zhaiyifan 《从零开始》 学习笔记

Posted by Pdog on 2018-05-13

zhaiyifan 《从零开始》 笔记

好像在看插件化还是组件化的资料的时候发现了zhaiyifan的博客，博客中有一个从零开始的系列。看了其中两篇都觉得很有干货，虽然已经是2年前（2016年）的东西了，但是我觉得应该值得一看，顺便自己做一个笔记记录下一些新学到的东西。

Todo: 如果有精力的话，我应该根据《从零开始》建立一个自己的github项目（without mvvm，dagger2）

1. 架构搭建篇

这篇讲了presentation - data - domain 的分层

我的理解是

  • domain 就是数据模型，
  • data就是一整套和数据打交道的容器（工具）
  • presentation 将数据和视图链接的工具

其中介绍了每层的测试相关的内容，但是我对这边不了解。

同时讲了 一些公共库和三方库，不复制一遍了

2. Gradle篇

这篇比较简单，列举一下标题

  • module 类型的区分 (application,library,java)
  • 版本号管理(ext属性使用)
  • 依赖管理(亮点的是针对debug和release定义了不同的map)
  • 签名管理（亮点是使用assert语句确保relsease时避免错误打包）
  • 其他(java8，Split APK)
  • Module的arr依赖

3. MVPVM

介绍了一种MVP和MVVM互相结合的架构

其实MVP是MVC的一个wrap，C层仍然可以在那里，代替View处理点击事件、数据绑定、扮演ListView的观察者，

一直以来都说MVP中的P取代了MVC中的C，但是我觉得这种说法更加对。MVP是MVC的一个wrap

另外在MVP的基础上引入了ViewModel，由于我没有使用也不准备学习MVVM，所以对这一块随意看了一下直接跳过了。

NO Presenter

我们不能永远理想化地去选择所谓最好的设计，在现实的必要情况下，我们要敢于舍弃，最合适的设计才是最好的设计。

非常赞同敢于舍弃这个观点，在我以后的架构设计中，应该避免这种强制性绑定V和P的做法（有V就必须有P，太啰嗦了）。

有趣的事作者大佬说有空的话我最近会在GitHub上写一下demo ，但是两年过去了好像( From 0 )并没有嘛（笑）

4. Dagger2

这篇介绍了一些依赖注入的内容，以及Dagger2。

虽然我在项目中没有使用过Dagger2，但是我还是对它有一定了解的。

博客中的有一句话似乎有些解释了我一直以来的疑问”Dagger2 的好处”

Dagger2解决了这些问题，帮助我们管理实例，并进行解耦。new只需要写在一个地方，getInstance也再也不用写了。

这句话new只需要写在一个地方，我需要实践的时候帮助理解一下

另外还有说到的一点就是

想想你所有可能在多个地方使用的类实例依赖，比如lbs服务，比如你的cache，比如用户设置，比起getInstance，比起new，比起自己用注释去注明必须维持这种先后关系

也需要以后用到dagger2的时候体会吧

5. Repository层 (上)

介绍了一下retrofit ，这里就直接跳过了。

另外还有一个二次组合的解析的思路，思路很不错学习了，但是没有什么实际意义。

Repository层组装实现介绍了repository的两种封装方式。但是因为太繁琐，我应该不会去这样用（笑）。

6. Repository层 (上)

介绍了数据库Realm的使用，内存缓存，直接跳过了

统一的异常处理

文章中提到 “作为Repository层，不应该去处理任何异常，应该抛给上层”，

7. DataBinding入门,8. DataBinding高级

我对DataBinding没有兴趣所以粗略的过了一遍。

9. 接口设计

这篇介绍了接口的设计。以后有机会可以当作一个参考文档来实现。

  • 接口设计

    其中提到了使用 http header来放置通用性的参数，觉得很不错，这样可以让我们的请求体(或响应体)更加关注内容本身。

  • Scheme

    另外提到了Scheme，暂时没有用到对这一块不是很了解，但是大概明白是什么东西，有机会需要学习

  • 回传 or 状态码

    更应该使用回传，而不是状态码，这样前端更好可以额外的信息进行一些状态的改变。文中举了一个非常直观的例子：用户点击关注后发生的事。

  • 模块和页面

    比较清晰的阐述了接口和页面对应的关系，更多的是规范的形式

  • 分页信息

    1. 后端需要返回页码
    2. 提到了一个让后端返回一个has_more字段，这样用来避免最后一次不必要的请求

  • 配置和空字段

    讲了一些数据校验的事情，前后端都需要对数据保持警惕，而不是完全依赖某一方。同时“保持灵活”。

10. React Native & Redux

关于RN在开发时的用处。这个应该另外深入

11. 组件化

比较了组件化和插件化的区别，并且提出了一些插件化的问题

同时减少了组件化的一些内容和坑，我就不一一复制了。
